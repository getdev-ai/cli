# CI/CD — GitHub Actions setup

Two workflows: `ci.yml` (every push/PR — correctness) and `release.yml` (tags — artifacts).
Target: full CI matrix under **10 minutes** via sccache + dependency caching.

## `.github/workflows/ci.yml`

Jobs (all required for merge):

| Job | Runner(s) | What |
|---|---|---|
| `fmt` | ubuntu | `cargo fmt --check` |
| `clippy` | ubuntu | `cargo clippy --workspace --all-targets -- -D warnings` |
| `test` | ubuntu, macos, windows | `cargo test --workspace` + corpus integration tests |
| `deny` | ubuntu | `cargo deny check` (licenses + RustSec advisories) |
| `coverage` | ubuntu | `cargo llvm-cov --workspace` — fail under 80% on library crates |
| `bench` | ubuntu (pinned runner type) | criterion against perf budgets; regression = red |
| `docs` | ubuntu | `cargo doc --no-deps` warnings-clean; link check on `docs/*.md` |

Reference skeleton:

```yaml
name: ci
on:
  push: { branches: [main] }
  pull_request:
permissions:
  contents: read
env:
  CARGO_TERM_COLOR: always
  RUSTC_WRAPPER: sccache
  SCCACHE_GHA_ENABLED: "true"
jobs:
  test:
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable   # respects rust-toolchain.toml
      - uses: mozilla-actions/sccache-action@v0.0.9
      - uses: Swatinem/rust-cache@v2
      - run: cargo test --workspace
      - run: cargo test -p getdev-cli --test corpus
  # fmt / clippy / deny / coverage / bench jobs follow the same pattern
```

Notes:

- **Toolchain** comes from `rust-toolchain.toml` — never hardcode a version in the workflow.
- **aarch64** is not in the PR matrix (cost); it's covered by release-artifact smoke tests.
- The **bench** job runs on one pinned runner class; treat >10% regressions as failures but
  expect noise — compare against a stored baseline (`criterion --save-baseline`), don't
  assert absolute wall-clock on shared runners.
- Corpus tests must not hit the network: `real` tests run against a recorded registry
  cache / local mock, and CI sets `GETDEV_OFFLINE=1` for everything except the dedicated
  registry-client tests (which use mocks anyway). A CI run must be hermetic.
- Windows job is a first-class citizen (CRLF, path separators, ANSI) — never mark it
  optional "temporarily".

## `.github/workflows/release.yml`

**Generated by `cargo dist init`** from `dist-workspace.toml` — do not hand-edit the
generated sections; regenerate on cargo-dist upgrades and review the diff. Trigger: tags
matching `v*`. Full behavior documented in [RELEASING.md](RELEASING.md).

Required permissions on the release workflow:

```yaml
permissions:
  contents: write        # create the release, upload artifacts
  id-token: write        # cosign keyless signing (OIDC)
  attestations: write    # SLSA provenance
```

Custom (non-generated) publish steps appended to it: npm publish, Homebrew tap push,
Scoop manifest bump, ordered `cargo publish`. Secrets these need are listed in
[RELEASING.md](RELEASING.md#required-repository-secrets); grant them to the workflow via a
`release` **environment** with required-reviewer protection, so a compromised PR can't
exfiltrate publish tokens.

## `.github/dependabot.yml`

```yaml
version: 2
updates:
  - package-ecosystem: cargo
    directory: /
    schedule: { interval: weekly }
    groups:
      minor-and-patch:
        update-types: ["minor", "patch"]
  - package-ecosystem: github-actions
    directory: /
    schedule: { interval: weekly }
```

`cargo-deny` in CI is the security backstop between Dependabot runs.

## Repository settings (one-time)

- **Branch protection on `main`**: require PRs; required checks = fmt, clippy, test×3, deny,
  coverage; require linear history (merge queue or squash); no force-push.
- **DCO enforcement**: enable the DCO check app (or the `dco` GitHub Action) as a required check.
- **Actions hardening**: pin third-party actions by SHA; default `permissions: contents: read`
  at workflow level; disable `pull_request_target` usage entirely (getdev scans untrusted
  repos — its own CI must not be an injection vector).
- **Private vulnerability reporting**: enabled (pairs with [../SECURITY.md](../SECURITY.md)).
- **Tag protection**: `v*` tags restricted to maintainers.

## Local parity

Everything CI runs is runnable locally with stock cargo commands (see
[BUILDING.md](BUILDING.md#lint--hygiene-ci-denies-warnings)) — if it's green locally on your
OS, the only usual surprises are Windows paths and snapshot drift.
